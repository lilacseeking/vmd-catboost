## 1. 问题背景：配电网物资需求预测

### 1.1 为什么这个问题重要？

配电网公司每年需要采购大量物资——10KV电缆、变压器、避雷器等。采购太多，库存积压、资金浪费；采购太少，工程延期、停电风险。**准确的需求预测直接关系到几千万的采购预算。**

### 1.2 为什么这个问题难？

物资需求不是凭空产生的，它受多种因素驱动：

| 物资 | 主要影响因素 | 数据特征 |
|------|-------------|---------|
| 10KV电缆 | 工程投资量、历史需求量、负荷增长、设备成本 | 冬季(12-2月)需求为0，非冬季10-30单位 |
| 柱上变压器 | 负荷增长、工程投资、历史需求、设备成本 | 冬季为0，非冬季3-18套，高波动+随机尖峰 |
| 避雷器 | **雷击次数、台风天数、暴雨次数**、负荷增长 | 干季(11-3月)为0，雨季**双峰**(5月+8月) |

注意避雷器的特殊性：它的需求主要由**天气**驱动，而不是投资。雷击多→避雷器损坏多→更换需求大。这就是为什么避雷器的影响因素排第一的是"雷击次数"而不是"投资量"。

### 1.3 为什么单一模型不够？

思考这个问题：给你过去48个月的数据，预测未来12个月的需求。你能用什么方法？

- **只用回归模型(如CatBoost)**：把"投资量=100万，雷击=0.8，月份=6月"映射到一个需求数字。但这样**完全忽略了时间顺序**——把6月的数据和12月的数据一视同仁，模型不知道"上个月需求高，这个月可能也高"。

- **只用时间序列模型(如LSTM)**：只看历史需求的变化趋势。但这样**忽略了外部驱动因素**——模型不知道"这个月雷击特别多，所以需求应该特别高"。

- **理想方案**：既要时序建模能力（捕捉需求的季节性、趋势性），又要因果推理能力（利用投资、天气等因素）。这就是VMD-LSTM-CatBoost的设计动机。

## 2. 数据：模拟但不虚假

### 2.1 为什么不直接用真实数据？

真实配电网数据涉及商业机密和安全规范，学术界常用**符合物理规律和业务逻辑的模拟数据**做方法验证。关键原则：**数据可以模拟，但生成规则必须合理。**

### 2.2 数据生成的核心逻辑

以避雷器为例，理解"双峰策略"：

```python
# 5月峰值: 雷雨季刚开始，大量避雷器需要更换
peak_may = exp(-0.5 * ((month - 4) / 1.0)^2) * 22

# 8月峰值: 台风+雷暴高发期，避雷器损坏率最高
peak_aug = exp(-0.5 * ((month - 7) / 0.85)^2) * 26

# 干季(11-3月): 几乎没有雷击，需求为0
```

这里用了**高斯函数**（正态分布的密度函数）来模拟**季节性峰值**。5月的峰值中心在μ=4（对应5月），8月的峰值中心在μ=7（对应8月），且8月峰值更高(26>22)更窄(σ=0.85<1.0)，反映了台风季避雷器需求更集中、更紧急的特点。

### 2.3 共享因子 vs 独立因子

注意代码中有一个重要设计：

```python
rng_shared = np.random.RandomState(42)   # 共享因子用主种子
rng_cable  = np.random.RandomState(43)   # 电缆独立随机状态
rng_trans  = np.random.RandomState(55)   # 变压器独立随机状态
rng_arr    = np.random.RandomState(45)   # 避雷器独立随机状态
```

为什么这样设计？**负荷增长量**是统一的——同一个地区，同一个月，负荷增长对所有物资是一样的。但每种物资的**随机噪声**应该独立——电缆的需求波动不应该影响避雷器的需求。使用独立的`RandomState`确保修改一种物资的数据生成逻辑时，其他物资的数据完全不受影响。

### 2.4 数据的多频率结构

为什么电缆和变压器数据要叠加多个正弦波？

```python
cable_annual  = sin(2π * t/12) * 5.0    # 年周期：每年一个波峰
cable_semi    = sin(4π * t/12) * 3.0    # 半年周期：每年两个波峰
cable_quarter = cos(8π * t/12) * 2.0    # 季度周期：每年四个波峰
cable_yearly  = sin(n * 0.22) * 3.5      # 年际波动：~2.4年的长周期
cable_trend   = 0.2 * t                  # 长期增长趋势
```

这对应了VMD分解的物理直觉：真实世界的序列不是单一频率的，而是**多种频率成分的叠加**。年周期（12个月）、半年周期（6个月）、季度周期（3个月）、年际波动（~28个月）、长期趋势——VMD正是要把这些成分分开。

## 3. 评估指标：如何判断模型好不好

### 3.1 四个核心指标

```
MSE  = (1/n) * Σ(y_true - y_pred)²    # 均方误差，对大误差敏感（平方放大）
RMSE = √MSE                             # 均方根误差，和原始数据同单位
MAE  = (1/n) * Σ|y_true - y_pred|      # 平均绝对误差，对异常值不敏感
R²   = 1 - Σ(y_true-y_pred)²/Σ(y_true-y_mean)²  # 决定系数，越接近1越好
```

### 3.2 为什么R²是最重要的指标？

MSE和RMSE的绝对值受数据尺度影响——避雷器需求在55~105之间，RMSE=18看起来很大；电缆需求在10~30之间，RMSE=6.8看起来很小。**无法跨物资比较。**

R²消除了尺度影响：R²=0.8意味着模型解释了80%的需求变化。避雷器R²=0.83和电缆R²=0.68可以直接比较——避雷器预测更准。

**但R²有陷阱**：如果真实值几乎不变（方差≈0），即使预测很差R²也可能接近1。这时要结合RMSE和MAE一起看。

### 3.3 解读本系统的结果

| 模型 | 电缆R² | 变压器R² | 避雷器R² | 均值R² |
|------|:---:|:---:|:---:|:---:|
| CatBoost | 0.230 | 0.459 | 0.547 | 0.412 |
| VMD-CatBoost | 0.137 | 0.261 | 0.405 | 0.268 |
| **VMD-LSTM-CatBoost** | **0.677** | **0.645** | **0.830** | **0.717** |
| VMD-LSTM(直接求和) | -0.042 | -0.302 | -0.009 | -0.118 |
| VMD-SVR | -2.521 | -2.702 | 0.370 | -1.618 |

几个关键观察：
- **VMD-LSTM-CatBoost全面领先**，均值R²=0.717，比第二名CatBoost(0.412)高出74%。
- **VMD-CatBoost反而不如纯CatBoost**——VMD分解出的IMF用CatBoost直接回归效果不好，因为IMF是时间序列，CatBoost不擅长纯粹的时间预测。
- **VMD-LSTM直接求和几乎无效**(R²负值)——说明CatBoost融合层是**不可或缺**的。LSTM单独预测每个IMF再求和，误差会累积放大。
- **VMD-SVR最差**——SVM不太适合小样本高噪声的时间序列外推。

## 4. 模型一：CatBoost —— 梯度提升树的巅峰

### 4.1 梯度提升树(Gradient Boosting)的核心思想

想象你在玩飞镖。第一镖偏左了10cm，第二镖你就应该向右调整。梯度提升树就是在不断"修正上一棵树的错误"：

```
预测值 = 树1(特征) + 树2(特征, 树1的残差) + 树3(特征, 树2的残差) + ...
```

每棵新树不直接预测目标值，而是预测**当前模型的残差**（真实值-当前预测值）。每棵树的预测乘以一个小的学习率再加到模型中，像"小步快跑"一样逼近目标。

### 4.2 CatBoost相比XGBoost/LightGBM的优势

| 特性 | XGBoost | LightGBM | **CatBoost** |
|------|---------|----------|-------------|
| 处理类别特征 | 需要手动编码 | 需要手动编码 | **自动处理** |
| 过拟合防护 | L1/L2正则化 | L1/L2正则化 | **Ordered Boosting** |
| 默认参数性能 | 需要调参 | 需要调参 | **开箱即用** |
| 小样本表现 | 容易过拟合 | 中等 | **最好** |

**Ordered Boosting** 是CatBoost的核心创新。传统梯度提升用全部数据计算梯度，这会导致**梯度估计偏差**（用同一批数据既训练模型又估计梯度）。CatBoost对每个样本只用"排在它前面"的样本计算梯度，避免了信息泄露。这在**小样本场景（48个训练点）**尤为重要。

### 4.3 为什么CatBoost基线只用4个因子？

```python
# 基线模型只用原始4因子，不用特征工程 → 凸显VMD-LSTM-CatBoost的时序建模优势
X_tr_raw = X_train_factors[:, :4].copy()  # 只取前4列：原始因子
```

这是故意的。如果CatBoost也用lag特征+月份编码，它和VMD-LSTM-CatBoost的差距会缩小，但我们想知道的是：**VMD+LSTM带来的时序建模能力到底值多少？** 答案是：R²从0.412提升到0.717。

## 5. 模型二：VMD —— 信号分解的利器

### 5.1 什么是VMD？

**变分模态分解(Variational Mode Decomposition, VMD)**是2014年提出的一种**完全非递归**的信号分解方法。它的核心思想是：任何复杂的信号都可以分解为若干个**本征模态函数(Intrinsic Mode Function, IMF)**之和，每个IMF是一个**窄带信号**（频率集中在一个小范围内）。

### 5.2 VMD vs EMD vs 傅里叶变换

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 傅里叶变换 | 信号=多个正弦波之和 | 数学基础坚实 | 只能处理平稳信号 |
| EMD(经验模态分解) | 递归地提取IMF | 自适应，不需预设K | 模态混叠，对噪声敏感 |
| **VMD** | 变分优化求解IMF | **抗噪，无模态混叠** | 需要预设K |

EMD像一个手工匠人，一层一层地剥离信号的"外皮"；VMD像一个优化器，同时找出所有频率成分的最优分解。VMD对噪声更鲁棒，更适合我们这种带噪声的模拟数据。

### 5.3 VMD的核心参数：K和α

```
K  = 分解出的IMF个数
α  = 带宽约束参数（惩罚因子）
```

- **K太小**（如K=2）：欠分解，不同频率的成分混在一个IMF里
- **K太大**（如K=8）：过分解，同一个频率成分被拆到多个IMF里，产生"假模态"
- **α太小**（如α=500）：每个IMF带宽太宽，频率分离不干净
- **α太大**（如α=5000）：每个IMF带宽太窄，可能丢失信息

### 5.4 如何自动选择K？

本系统实现了一个基于**中心频率分离度**的自动K选择：

```python
def vmd_optimize_k(signal):
    for k in [3, 4, 5, 6, 7, 8]:
        做VMD分解得到K个IMF的中心频率
        if 相邻中心频率的比值都 > 1.3:
            best_k = k  # 分解充分，尝试更大的K
        else:
            break      # 出现频率混叠，停止
```

直觉：如果K=5时，5个中心频率是[0.01, 0.05, 0.12, 0.28, 0.55]，相邻比值都>1.3，说明5个IMF频率分离得很好。如果K=6时出现了[0.01, 0.05, 0.12, 0.22, 0.25, 0.55]，其中0.22和0.25的比值=1.14<1.3，说明K=6过分解了，最优K=5。

### 5.5 不同物资的最优α

| 物资 | α | 原因 |
|------|:---:|------|
| 电缆 | 2000 | 多频率成分混合，标准α |
| 变压器 | 1500 | 高波动+尖峰，较低α让IMF能捕获更宽频带 |
| 避雷器 | 3000 | 纯净的双峰结构，高α让频率分离更精细 |

避雷器的需求模式最"干净"——就是两个高斯峰+趋势+少量噪声。高α值让VMD能更精细地分离趋势、年周期和噪声，从而提取出更纯净的双峰信号。

### 5.6 IMF的物理意义

以避雷器（K=3或4）为例：

```
IMF1 (最低频): 长期趋势 —— "避雷器需求在缓慢增长"
IMF2 (中频):   年周期/双峰 —— "每年5月和8月的需求高峰"
IMF3 (高频):   噪声 —— "随机波动，不可预测"
```

VMD把这三者分离开后，LSTM可以分别预测趋势（简单外推）、季节模式（周期性预测）和噪声（可以忽略），然后CatBoost再把这些预测和外部因子融合。这就是"分而治之"的思想。

## 6. 模型三：LSTM —— 时序建模的核心

### 6.1 为什么需要LSTM？普通RNN不行吗？

传统RNN（循环神经网络）在处理长序列时会遇到**梯度消失/爆炸**问题。想象你在读一本100页的小说，读到第100页时，还能清晰记得第1页的内容吗？RNN做不到。

LSTM（长短期记忆网络）通过一个精巧的**门控机制**解决了这个问题：

```
遗忘门: "之前的记忆中，哪些该忘记了？"
输入门: "新的信息中，哪些该记住？"  
输出门: "当前该输出什么？"
```

这三个门都是可学习的神经网络层，LSTM自己学会"什么时候该记住，什么时候该忘记"。

### 6.2 MultiFeatureLSTM vs SingleFeatureLSTM

本系统用了两种LSTM：

**MultiFeatureLSTM**（用于残差分量）：
```
输入: [残差IMF值, 因子1, 因子2, 因子3, 因子4] — 每个时间步5个特征
结构: 2层LSTM, hidden_size=8, dropout=0.25
输出: 下一个时间步的残差IMF预测值
```
为什么残差分量要用多特征？因为**趋势分量受外部因子影响最大**——投资增加→需求趋势上升。所以给LSTM同时看趋势值和因子值，让它学习"因子如何驱动趋势变化"。

**SingleFeatureLSTM**（用于模态分量）：
```
输入: [模态IMF值] — 每个时间步1个特征
结构: 2层LSTM, hidden_size=6, dropout=0.25
输出: 下一个时间步的模态IMF预测值
```
为什么模态分量只用单特征？因为**季节性主要是时间自身的函数**——5月就是5月，雷击多就是雷击多，这个模式是时间本身决定的，不太受投资等因子影响。

### 6.3 滑动窗口：如何构建训练样本？

```python
# seq_len=6, stride=1
# 原始序列: [x0, x1, x2, x3, x4, x5, x6, x7, ..., x47]
# 样本1: X=[x0...x5], y=x6
# 样本2: X=[x1...x6], y=x7   ← 与样本1重叠5个时间步!
# ...
# 样本42: X=[x41...x46], y=x47
```

stride=1意味着相邻样本之间只差一个时间步，总共产生48-6=42个训练样本。对比stride=2（只有21个样本），**样本量翻倍**，在数据稀缺的场景下非常关键。代价是样本之间有很强的相关性（overlap），但在小数据场景下**增加样本量的收益远大于样本相关的代价**。

### 6.4 LSTM的训练技巧

```
学习率调度(ReduceLROnPlateau): loss不降时自动减半学习率
早停(Early Stopping): validation loss连续60轮不降就停止训练
梯度裁剪(Gradient Clipping): 限制梯度最大范数为1.0
Dropout(0.25): 训练时随机丢弃25%的神经元连接，防止过拟合
Weight Decay(1e-4): 权重衰减，L2正则化
```

这些技巧的目标都是同一个：**用42个样本训练神经网络而不严重过拟合**。没有这些正则化手段，LSTM会在训练集上表现极好（loss≈0）但在测试集上表现极差——这就是过拟合。

## 7. 核心模型：VMD-LSTM-CatBoost 三阶段混合架构

### 7.1 为什么是这个组合？

这是整个系统最核心的设计决策。让我用一个比喻来解释：

**VMD = 分诊台**：把复杂的混合信号拆成"趋势科"、"季节科"、"噪声科"
**LSTM = 各科专家**：每个专家只看自己擅长的部分做预测
**CatBoost = 主任医师**：综合所有专家的意见，结合化验单（外部因子），做出最终诊断

如果跳过VMD直接让LSTM预测原始信号→LSTM要同时学习趋势、季节、噪声，难度大、容易过拟合。如果跳过CatBoost直接把LSTM预测加起来(VMD-LSTM直接求和)→误差也直接累加，精度差。**三个环节缺一不可。**

### 7.2 完整数据流

```
输入: 48个月的需求量序列 + 4个外部因子序列
                    │
    ┌───────────────┴───────────────┐
    │         VMD分解(K自动优化)     │
    │  需求量 → [IMF1, IMF2, ..., IMFk] │
    └───────────────┬───────────────┘
                    │
         ┌──────────┴──────────┐
         │   IMF相关性筛选      │
         │  剔除与原始序列相关度<0.1的噪声IMF │
         └──────────┬──────────┘
                    │
    ┌───────────────┴───────────────┐
    │         IMF外推至测试期         │
    │  趋势分量: 线性回归外推          │
    │  模态分量: persistence(重复末值) │
    └───────────────┬───────────────┘
                    │
    ┌───────────────┴───────────────┐
    │         LSTM 时序预测           │
    │  残差分量 → MultiFeatureLSTM   │
    │  模态分量 → SingleFeatureLSTM  │
    │  每个IMF独立训练一个LSTM        │
    └───────────────┬───────────────┘
                    │
    ┌───────────────┴───────────────┐
    │       CatBoost 融合层          │
    │  N个LSTM预测值 + 8维特征        │
    │  → 最终需求量预测               │
    └───────────────────────────────┘
```

### 7.3 为什么CatBoost融合比直接求和好？

直接求和的逻辑是：VMD满足Σ(IMF_i) = 原始信号，所以Σ(LSTM预测的IMF_i)应该等于预测的需求量。这在理论上没问题，但实际上：

1. **每个LSTM都有预测误差**，直接求和意味着误差也直接求和
2. **IMF之间的预测误差不是独立的**——某个LSTM低估了，另一个可能也低估了（因为训练数据是同一段序列）
3. **外部因子提供了额外信息**——"这个月投资量翻倍了"这个信息在直接求和中完全没用上

CatBoost融合的做法是：把N个LSTM预测值当作N个特征，加上投资、天气等外部因子，让模型学习"如何最优地加权组合这些预测"。比如它可能学到："避雷器的5月峰值主要由IMF2驱动，要给它更高的权重；但8月峰值受台风因子影响很大，要更依赖外部因子"。

## 8. 消融实验：证明每个组件都有用

### 8.1 什么是消融实验(Ablation Study)？

如果你说"A+B+C的组合效果最好"，别人可以问："真的需要A吗？B+C行不行？" 消融实验就是系统地**移除一个组件**，观察性能下降多少。下降越多，说明这个组件越重要。

### 8.2 本系统的消融设计

| 模型 | VMD | LSTM | CatBoost融合 | 含义 |
|------|:---:|:---:|:---:|------|
| CatBoost | ✗ | ✗ | ✗ | 纯回归基线 |
| VMD-CatBoost | ✓ | ✗ | ✓ | 有分解无LSTM |
| VMD-LSTM-CatBoost | ✓ | ✓ | ✓ | 完整架构 |
| VMD-LSTM(直接求和) | ✓ | ✓ | ✗ | 移除CatBoost融合层 |
| VMD-SVR | ✓ | ✗ | SVR替代 | 用SVM取代梯度提升 |

### 8.3 结论

| 消融 | R²变化 | 结论 |
|------|:---:|------|
| 移除VMD+LSTM+CatBoost融合 | 0.717→0.412 | 三件套整体贡献+0.305 |
| 移除LSTM（保留VMD+CatBoost） | 0.717→0.268 | **LSTM是最关键组件**，贡献最大 |
| 移除CatBoost融合（保留VMD+LSTM） | 0.717→-0.118 | **融合层不可或缺**，直接求和完全失效 |
| 用SVR替代CatBoost | 0.717→-1.618 | CatBoost融合远超SVR |

**核心结论**: LSTM是时序建模的灵魂，CatBoost融合层是串联信息的关键。二者缺一不可。

## 9. 关键技术细节：Look-Ahead Bias与RNG隔离

### 9.1 Look-Ahead Bias：时间序列预测的"死罪"

**Look-Ahead Bias（前视偏差）**是指在训练时不小心使用了**未来的信息**。在时间序列预测中，这是最严重的错误之一——它会让你的模型看起来超好，但在真实场景中完全失效。

本系统在三个地方严防Look-Ahead Bias：

**1. VMD仅对训练集分解**
```python
# ❌ 错误做法：对全部60个月做VMD，然后用前48个月训练
u_full = VMD(all_60_months)  # 后12个月的信息泄露到IMF中！
# ✅ 正确做法：只对训练集48个月做VMD
u_train = VMD(train_48_months)
```

**2. IMF外推代替直接分解测试集**
```python
# ❌ 错误做法：对测试集直接做VMD
u_test = VMD(test_12_months)  # 测试集的VMD用了测试集数据！
# ✅ 正确做法：基于训练集的IMF模式外推
u_test = extrapolate_imfs(u_train, n_test=12)
```

**3. 时序分割保证因果性**
```python
# 训练集: 2020.01 - 2023.12 (前48个月)
# 测试集: 2024.01 - 2024.12 (后12个月)
# 绝不混洗！时间顺序不可打破！
```

### 9.2 RNG隔离：为什么每个物资需要独立随机状态

```python
rng_cable  = np.random.RandomState(42 + 1)  # 不是 np.random!
rng_trans  = np.random.RandomState(42 + 13)
rng_arr    = np.random.RandomState(42 + 3)
```

如果你修改了电缆的数据生成逻辑（增加了几个`np.random.randn()`调用），全局随机状态就会改变，导致变压器和避雷器的数据也跟着改变——即使它们的生成逻辑完全没有修改！这会让你无法判断"性能变化是因为修改了电缆的逻辑，还是因为变压器的数据被随机改变了"。

使用独立的`RandomState`对象，每个物资的随机数序列完全独立，互不干扰。这是**可复现研究**的基本要求。

### 9.3 VMD的随机状态隔离

```python
def run_vmd_lstm_catboost(...):
    np.random.seed(RANDOM_SEED)  # 每个函数入口重置随机种子！
```

为什么需要这个？vmdpy库的VMD函数内部可能使用全局`np.random`进行初始化。如果不重置，前一个物资的VMD会消耗随机数，影响后一个物资的VMD结果。这个bug非常隐蔽——数据完全一样，但VMD分解结果不同，最终导致预测结果不可复现。

## 10. 特征工程：让模型看到更多信息

### 10.1 本系统使用的特征

| 特征类别 | 特征 | 维度 | 说明 |
|---------|------|:---:|------|
| 原始因子 | investment, load_growth, etc. | 4 | 直接驱动需求的外部变量 |
| 滞后特征 | lag1, lag2 | 2 | t-1和t-2时刻的需求量 |
| 时间编码 | month_sin, month_cos | 2 | 月份信息的循环编码 |
| **总计** | | **8** | |

### 10.2 为什么用sin/cos编码月份而不是直接用1-12？

```python
# ❌ 直接用月份数字
month = [1, 2, 3, ..., 12]
# 问题：12月和1月在数值上差11，但在时间上只差1个月！
#      模型以为12和1很远，实际上它们相邻。

# ✅ 用sin/cos编码
month_sin = sin(2π * month / 12)  # 12月→0, 1月→0.5, 6月→0
month_cos = cos(2π * month / 12)  # 12月→1, 1月→0.87, 6月→-1
# 12月和1月的编码值非常接近！正确地反映了它们的时间邻近关系。
```

sin/cos编码把圆环（12个月是循环的）映射到一个二维平面上，12月和1月在这个平面上是相邻的。这是处理**循环特征**（月份、星期、小时、风向等）的标准做法。

### 10.3 滞后特征为什么重要？

`lag1 = 上个月的需求量`

这是一个非常强的特征。如果上个月需求是20，这个月大概率也是20附近。一阶自相关（自己和自己的滞后值的相关性）在大多数经济时间序列中都非常显著。

但我们只用了lag1和lag2，没有用更多（lag3, lag4...）。为什么？两个原因：
1. **自由度限制**：48个训练样本，特征太多→过拟合风险
2. **信息衰减**：lag1已经很接近当前值了，lag3/lag4的边际贡献很小

## 11. 超参数优化：20轮迭代的教训

### 11.1 什么有效？（按贡献排序）

| 排名 | 优化项 | R²提升 | 教训 |
|:---:|------|:---:|------|
| 1 | 数据多频率增强 | +0.06 | **数据质量 > 模型复杂度** |
| 2 | 避雷器双峰强化+降噪 | +0.04 | 信号清晰度决定预测上限 |
| 3 | stride=1 (42样本) | +0.03 | 小数据场景下样本量很关键 |
| 4 | RNG隔离 | 稳定性 | 没有隔离就没有可靠迭代 |
| 5 | VMD alpha按物资 | +0.01 | 微调有收益但不大 |

### 11.2 什么无效？

| 尝试 | 结果 | 原因 |
|------|------|------|
| 3层LSTM | 不升反降 | 42个样本，参数越多越容易过拟合 |
| CatBoost深度增加(depth 6→7) | 过拟合 | 融合训练样本更少(~42个) |
| dropout 0.25→0.3 | R²暴跌 | 过度正则化，模型学不到东西 |
| Adam→AdamW | 退化 | 不是优化器的问题 |
| 双向LSTM | 不适合 | 预测不能用未来信息 |
| SVR替代CatBoost | R²负值 | SVR不适合高维小样本场景 |

### 11.3 最重要的教训

**1. 小数据下，简洁就是美。** 2层LSTM > 3层，hidden=8 > hidden=12。数据量决定了模型复杂度的上限，强行加复杂度只会过拟合。

**2. 数据工程远重于模型调参。** 把电缆数据从简单年周期改成多频率叠加（年+半年+季度+年际），R²直接涨了6个百分点。这个收益比调任何超参数都大。

**3. 可复现性是优化的前提。** RNG隔离和VMD状态隔离看起来是"工程细节"，但没有它们，你根本无法判断哪次改动真的有效。很多研究员花几个月调参，最后发现是随机波动。

> **最后的话**: 一个好的预测系统不是模型的堆砌，而是**对问题的深刻理解 + 对每种方法适用场景的清醒认知 + 严谨的工程实践**。VMD适合分解多频率信号，LSTM适合建模时序依赖，CatBoost适合融合异构特征——把它们放在一起不是因为它们"流行"，而是因为在这个具体问题中，每个组件恰好解决了整体难题中的一个子问题。理解**为什么**永远比知道**怎么做**更重要。
